# Telegram бот для продажи VPN с автоматической оплатой через ЮKassa

---

Telegram VPN Bot - это мощный и многофункциональный Telegram бот для автоматизации продажи VPN-подписок с интеграцией платежной системы ЮKassa. Бот поддерживает протоколы VLESS и V2Ray (XRay), обеспечивает полностью автоматическую выдачу конфигурационных ключей после успешной оплаты, имеет встроенную реферальную систему с начислением 25% от платежей приглашенных пользователей, а также предоставляет обширную административную панель для полного контроля над бизнесом.

Быстрая установка

Автоматическая установка одной командой

```bash
wget -O - https://raw.githubusercontent.com/thetemirbolatov/Telegram-Bot-VPN/main/install.sh | sudo bash
```

Эта команда автоматически:

1. Установит Python 3 и pip если они отсутствуют
2. Установит все необходимые зависимости из requirements.txt
3. Создаст структуру директорий для бота
4. Настроит автозапуск через systemd (на Linux системах)
5. Создаст шаблоны конфигурационных файлов

После автоматической установки

1. Отредактируйте файл vpn.py и укажите:
   · TOKEN - токен вашего бота от BotFather
   · ADMIN_ID - ваш Telegram ID
   · YOOKASSA_SHOP_ID и YOOKASSA_SECRET_KEY в файле yookassa_integration.py
2. Запустите бота:
   ```bash
   sudo systemctl start vpnbot
   sudo systemctl enable vpnbot
   ```
3. Проверьте статус:
   ```bash
   sudo systemctl status vpnbot
   ```

Ручная установка и настройка

Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/thetemirbolatov/Telegram-Bot-VPN.git
cd Telegram-Bot-VPN
```

Шаг 2: Создание виртуального окружения (рекомендуется)

```bash
python3 -m venv venv
source venv/bin/activate  # для Linux/Mac
# или
venv\Scripts\activate  # для Windows
```

Шаг 3: Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

Содержимое requirements.txt:

```
pyTelegramBotAPI==4.14.0
requests==2.31.0
qrcode==7.4.2
Pillow==10.0.0
openpyxl==3.1.2
pandas==2.0.3
yookassa==3.0.0
```

Шаг 4: Создание необходимых директорий

```bash
mkdir -p backups exports keys_storage
```

Шаг 5: Настройка конфигурационных файлов

Отредактируйте vpn.py:

```python
# Конфигурация бота
TOKEN = 'ВАШ_ТОКЕН_ОТ_BOTFATHER'
ADMIN_ID = 123456789  # Ваш Telegram ID
CHANNEL_USERNAME = ""  # Имя канала для обязательной подписки (опционально)
VIDEO_INSTRUCTION_URL = "https://t.me/karachay_aj"  # Ссылка на видеоинструкцию
```

Отредактируйте yookassa_integration.py:

```python
YOOKASSA_SHOP_ID = "ВАШ_SHOP_ID"
YOOKASSA_SECRET_KEY = "ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"
```

Шаг 6: Создание systemd сервиса (для Linux)

Создайте файл /etc/systemd/system/vpnbot.service:

```ini
[Unit]
Description=Aj VPN Telegram Bot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/Telegram-Bot-VPN
Environment="PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"
ExecStart=/usr/bin/python3 /root/Telegram-Bot-VPN/vpn.py
Restart=always
RestartSec=10
StandardOutput=append:/var/log/vpnbot.log
StandardError=append:/var/log/vpnbot-error.log

[Install]
WantedBy=multi-user.target
```

Активируйте сервис:

```bash
sudo systemctl daemon-reload
sudo systemctl start vpnbot
sudo systemctl enable vpnbot
```

Шаг 7: Запуск бота вручную

```bash
python3 vpn.py
```

2. Нажмите "Создать магазин"

4. Получите shopId и Секретный ключ

Откройте файл yookassa_integration.py:

```python
# Для тестового режима (используйте тестовые ключи из личного кабинета)
YOOKASSA_SHOP_ID = "test_shop_id"
YOOKASSA_SECRET_KEY = "test_secret_key"
```


Реферальная система

Принцип работы

Реферальная система позволяет пользователям зарабатывать 25% от суммы каждого платежа приглашенных ими пользователей.

1. Пользователь подает заявку на участие через кнопку "Подать заявку на участие" в разделе "Рефералы"
2. Администратор одобряет заявку и пользователь получает уникальную реферальную ссылку
3. Пользователь распространяет свою реферальную ссылку
4. Когда новый пользователь регистрируется по реферальной ссылке и оплачивает подписку:
   · Реферер получает 25% от суммы платежа на свой баланс
   · Баланс отображается в разделе "Рефералы"
5. При достижении минимальной суммы (любая сумма больше 0) пользователь может подать заявку на вывод средств
6. Администратор обрабатывает заявку и выплачивает средства на указанные реквизиты

Администрирование реферальной системы

Администратор может:

· Просматривать список всех участников программы
· Добавлять пользователей в программу вручную
· Удалять пользователей из программы
· Просматривать и обрабатывать заявки на вывод средств
· Видеть историю всех заявок на вывод

Уведомления

Система отправляет следующие уведомления:

· Пользователю: о добавлении в программу с реферальной ссылкой
· Пользователю: о начислении бонуса за покупку реферала
· Пользователю: о статусе заявки на вывод
· Администратору: о новой заявке на участие
· Администратору: о новой заявке на вывод средств

Создание рассылки

Администратор может создавать сложные рассылки с поддержкой:

· Текстовых сообщений
· Фотографий с подписью
· Видео с подписью
· Документов с подписью
· Аудио и голосовых сообщений
· Inline-кнопок с ссылками (до 5 кнопок)

Процесс создания рассылки:

1. Нажмите "Рассылка" в админ-панели
2. Отправьте контент (текст, фото, видео, документ)
3. Укажите количество кнопок (0-5)
4. Для каждой кнопки укажите текст и URL в формате: Название кнопки | https://ссылка
5. Подтвердите отправку

Пользовательский интерфейс

Главное меню

Пользователь видит следующие кнопки в главном меню:

· Купить VPN | Продлить - покупка новой подписки или продление существующей
· Активировать промокод - ввод промокода для получения бесплатных дней
· Мои ключи - просмотр всех активных VPN ключей
· Поддержка - контакты для связи с поддержкой
· Рефералы - реферальная программа и баланс
· Видео инструкция - ссылка на видео по настройке
· Инструкция установки - текстовая инструкция с ссылками на приложения

Процесс покупки

1. Пользователь нажимает "Купить VPN | Продлить"
2. Выбирает сервер (Германия или Нидерланды)
3. Выбирает срок подписки (1, 3 или 6 месяцев)
4. Бот создает платеж в ЮKassa и отправляет ссылку для оплаты
5. Пользователь оплачивает через удобный способ
6. После успешной оплаты бот автоматически выдает VPN ключ:
   · Информация о подписке
   · Ключ в текстовом виде
   · QR-код для сканирования
   · Ссылки на скачивание приложений
   · Видеоинструкция

Продление подписки

Если у пользователя уже есть активная подписка:

1. Нажмите "Купить VPN | Продлить"
2. Выберите "Продлить [сервер]"
3. Выберите срок продления
4. Оплатите
5. Срок действия текущего ключа автоматически увеличится

Просмотр ключей

В разделе "Мои ключи" пользователь видит:

· Список всех активных ключей с датами окончания
· Каждый ключ в текстовом виде
· QR-код для каждого ключа
· Ссылки на приложения

Резервное копирование и восстановление

Автоматическое резервное копирование

Бот автоматически создает резервные копии перед каждым сохранением данных:

```python
backup_files = ['users_db.json', 'payments_db.json', 'servers_db.json', 
                'payment_methods.json', 'promo_codes.json']
for file in backup_files:
    if os.path.exists(file):
        shutil.copy2(file, f"{file}.backup")
```

Ручное создание резервной копии

Администратор может создать полную резервную копию в любой момент:

1. Нажмите "Создать резервную копию" в админ-панели
2. Бот создаст ZIP архив со всеми данными и отправит его в чат
3. Имя файла: backup_YYYYMMDD_HHMMSS.zip

Восстановление из резервной копии

1. Нажмите "Восстановить из копии" в админ-панели
2. Отправьте ZIP файл резервной копии
3. Бот выполнит безопасное слияние данных (merge):
   · Существующие данные не удаляются
   · Новые данные добавляются
   · Конфликты разрешаются в пользу новых данных

Безопасность

Защита данных

1. Токен бота: хранится в коде, рекомендуется использовать переменные окружения
2. Ключи ЮKassa: хранятся в отдельном файле, не передаются пользователям
3. Проверка подписки: опциональная проверка подписки на канал перед использованием
4. Админ-доступ: только пользователи с указанным ADMIN_ID имеют доступ к админ-панели
5. Резервное копирование: автоматические бэкапы перед каждым сохранением

Рекомендации по безопасности

1. Переменные окружения:
   ```python
   import os
   TOKEN = os.environ.get('VPN_BOT_TOKEN')
   ADMIN_ID = int(os.environ.get('ADMIN_ID', '0'))
   YOOKASSA_SHOP_ID = os.environ.get('YOOKASSA_SHOP_ID')
   YOOKASSA_SECRET_KEY = os.environ.get('YOOKASSA_SECRET_KEY')
   ```
2. Ограничение прав доступа:
   ```bash
   chmod 600 *.json
   chmod 600 vpn_bot.log
   chmod 700 backups/
   ```
3. Регулярное обновление:
   ```bash
   pip install --upgrade pyTelegramBotAPI yookassa
   ```

Решение проблем

Часто встречающиеся проблемы

Бот не запускается

Причина: ошибка в конфигурации или отсутствие зависимостей

Решение:

```bash
# Проверьте версию Python
python3 --version

# Установите зависимости
pip install -r requirements.txt

# Проверьте синтаксис
python3 -m py_compile vpn.py
```

Ошибка соединения с Telegram API

Причина: проблемы с сетью или блокировка

Решение:

```python
# Используйте прокси
from telebot import apihelper
apihelper.proxy = {'https': 'socks5://user:pass@host:port'}
```

Платежи не проходят

Причина: неверные ключи ЮKassa

Решение:

1. Проверьте SHOP_ID и SECRET_KEY в yookassa_integration.py
2. Убедитесь что используется правильный режим (test/live)
3. Проверьте логи ЮKassa в личном кабинете

Ключи не выдаются после оплаты

Причина: закончились доступные ключи на сервере

Решение:

1. Проверьте количество доступных ключей в админ-панели
2. Добавьте новые ключи через "Добавить ключи массово"
3. Проверьте логи на наличие ошибок

Реферальные бонусы не начисляются

Причина: проблема с referral_db.json или неправильная структура

Решение:

1. Проверьте наличие файла referral_db.json
2. Убедитесь что пользователь добавлен в allowed_users
3. Проверьте что связь relations существует
4. Проверьте логи на наличие ошибок реферальной системы

Восстановление после сбоя

```bash
# Остановите бота
sudo systemctl stop vpnbot

# Восстановите из последнего бэкапа
cp backups/backup_*.zip .
python3 -c "
from vpn import restore_from_backup
restore_from_backup('backup_20240115_143000.zip')
"

# Запустите бота
sudo systemctl start vpnbot
```
